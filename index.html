记录一次conda更新源造成的影响

Posted on 2020-07-24

前一阵子，假期基本属于结束了，于是决定开始写点东西。那会儿想到已经很久没有写过代码了，于是打算先更新一下conda的包。首先运行了一下conda updade --all，显示需要更新的包后，我等待其更新。
期间由于源是属于国外的默认源，速度极其缓慢。我决定替换为国内的TH源，将channel替换后，按照指示清除索引缓存conda clean -i。大功告成后，我再次输入命令来更新包，于是出现了不可描述的第一个问题:

引发了这个问题后，所有的有关conda的install、update指令等，都会卡在这个界面非常长的时间。Fine, 我开始在网上搜索解决方案。
第一个方案来源于CSDN:

  1. 首先，输入conda list -r查询之前的更新版本，然后选择一个之前的版本（出现问题之前），输入conda install --revision number之后等待回滚成功.
  2. 回退成功后，删除之前添加的所有channel.

OK，OK。我看这位老哥跟我的情况差不多，按照他的手法来试了试。我找到了比较早的一个历史版本，我原本还担心我的一些安装过的环境需要重新下，结果在执行revision时，直接给我抛出了错误:

好了，这下也没办法回退了。我在StackOverFlow上寻找一会儿吧。最终在这个页面找到了一些个解决方案（https://stackoverflow.com/questions/55527354/ ）。解决方案大致如下：

  • conda install anaconda 。 无用，我前面说过，install， upgrade均卡住。
  • conda update --all。 无用，我就是执行这个卡住的 :-)
  • conda install -c anaconda anaconda。无用
  • …

一天多的尝试下来，最后，我也没招了。怎么办呢，我重新看了看TH源添加channel那。我逆向操作一下吧，将原先添加的源都删除到只剩下默认源，然后重新执行conda clean -i。莫名其妙就好使了。可能是之前索引缓存没有清理好？还是channel添加有问题？记录一下吧。

使用github.io来记录自己的博客

Posted on 2020-06-13 | In 生活

决定使用github.io来记录博客

​ 2020年6月，我的本科生涯就要结束了。四年时光很短暂（当然我指的是腾讯云1块钱的学生机:-)）。同时，我在查看许多技术大牛的博文时，发现他们很多都使用github.io来分享自己的技术贴，我个人尝试了一番之后，发现若单纯分享自己的技术贴，使用该page完全足够了。

​ 本科期间，我也基本属于比较懒的那一类人，不爱更新动态，不爱发博文。在2020年的暑假，我决定做出一些改观。假期里，我已经开始读相关研究方向的论文，后续在我的blog更新自己的论文解读。若遇到一些技术上的困难，解决后需要及时记录。一些环境搭建，我认为也有必要记录一下。

​ 总的来说，本科期间漏掉了很多记录点滴的机会，我在硕士期间要好好把握。该page会记录一些技术上的见解，论文解读，环境搭建，竞赛和会议杂谈等。若后续购买了新的云服务器，我也会同步更新。我认为这是一种很好的记录生活的方式。

最后希望我这小菜鸡能够做出一些成绩。也希望相关领域的朋友能够相互认识一下，谢谢！
